Javscript: Data types and Variables

How does data types work in Javascript?

Data Types in JavaScript specify what kind of data can be stored and manipulated within a program.

There are six most common data types in JavaScript which are deivided into three categories: primitive, composite and special data types.

String, Number and Boolean are primitive data types. Object, Array and Function (which are types of objects) are composite data types. Data types like Undefined and Null are special data types.

Primitive data types can hold one value at a time, while composite data types can hold collections of values and more complex entities.

Node.js

What is Node.js? And how does one run files with it?

Node. js is primarily used for non-blocking, event-driven servers, due to its single-threaded nature. It's used for traditional web sites and back-end API services, but was designed with real-time, push-based architectures in mind.

Run Node.js scripts from the command line

The usual way to run a Node.js program is to run the node globally available command (once you install Node.js) and pass the name of the file you want to execute.

If your main Node.js application file is app.js, you can call it by typing:


 node app.js

While running the command, make sure you are in the same directory which contains the app.js file.

CRUD REST API

What is CRUD and REST?
CRUD

Create, Read, Update, and Delete - or CRUD - are 4 functions used to interact with database applications. Many programming languages and protocols have their own equivalent of CRUD, with very similar namings. SQL's CRUD equivilant is for example called Insert, Select, Update and Delete.

Functions
Name Description
Create Adds one or more new entries
Read Retrieves entries that match certain criteria (if there are any)
Update Changes specific fields in existing entries
Delete Entirely removes one or more existing entries

CRUD

Representational State Transfer - or REST - is the popular architectural style for software, used a lot in web APIs. It's defined by five design constraints that produce an application with specific properties, including performance, simplicitiy, and reliability.

Constraints
Term Description
Client-Server The client and server act independently.
Stateless The server does not record the state of the client.
Cacheable The server marks whether data is cacheable.
Uniform Interface The client and server interact in a uniform and predictable way. An important aspect of this is that the server exposes resources.
Layered System The application behaves the same regardless of any intermediaries between the client and server.

GET routes in Express

How does GET routes in Express work?

Routing refers to determining how an application responds to a client request to a particular endpoint, which is a URI (or path) and a specific HTTP request method (GET, POST, and so on).

Each route can have one or more handler functions, which are executed when the route is matched.

Route definition takes the following structure:

app.METHOD(PATH, HANDLER)
  • app is an instance of express.
  • METHOD is an HTTP request method, in lowercase.
  • PATH is a path on the server.
  • HANDLER is the function executed when the route is matched.

Functional loop methods

How does looping using the functional loop methods like map, filter etc work?
.map()

the map() method creates a new array populated with the results of calling a provided function on every element in the calling array.

map calls a provided callback function once for each element in an array, in order, and constructs a new array from the results. callback is invoked only for indexes of the array which have assigned values.


.filter()

The filter() method creates a new array with all elements that pass the test implemented by the provided function.

filter() calls a provided callback function once for each element in an array, and constructs a new array of all the values for which callback returns a value that coerces to true. callback is invoked only for indexes of the array which have assigned values; it is not invoked for indexes which have been deleted or which have never been assigned values. Array elements which do not pass the callback test are skipped, and are not included in the new array.


.reduce()

Just like .map(), .reduce() also runs a callback for each element of an array. The difference here is that reduce() passes the result of this callback from one array element to the other.
